PHP中的二进制安全

5月 22, 2018 发布在 PHP

我们在看PHP的文档的时候，经常会看到一些方法被标注是二进制安全的，那么什么是二进制安全，为什么有些方法是二进制安全的，有没有二进制不安全的方法？

二进制安全

一句话总结，所谓二进制安全，就是你传一个任意二进制数据，方法都能正确执行。

其实我们都知道，程序中的字符串会按照相应的编码解析成计算机能够识别的二进制编码串。不同的编程语言对于字符串会有不同的识别格式。比如C，会将\0作为字符串的结束符。二进制不安全也是由此产生，如果一个字符串二进制码含有这样的特殊字符，程序的字符串方法在处理这些特殊字符的时候能够正确处理，那么这个方法就是二进制安全的。PHP的底层解析引擎是C语言，由于C语言的这些特性，导致PHP的在处理字符串时也要注意特殊字符引起的二进制安全问题。
